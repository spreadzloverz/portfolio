# Portfolio GitHub Pages — Xavier Talbot

## 1) Publication (GitHub Pages)
1. Crée un repo GitHub (ex: `portfolio`)
2. Uploade tout le contenu de ce dossier à la racine du repo
3. GitHub → Settings → Pages → **Deploy from a branch**
4. Branch: `main` / folder: `/root`
5. Ton site est en ligne (URL fournie par GitHub)

## 2) Où mettre les images
- Ajoute les images de chaque projet dans :
  `assets/img/projects/<slug>/`
- Exemple :
  `assets/img/projects/traits-identite/01.jpg`

## 3) Mettre les images dans les pages
Ouvre `projects/<slug>.html` et ajoute des `<img>` dans la section Galerie.

## 4) Projets à compléter
Les projets dont le titre est "INFORMATION NON PRÉSENTE" doivent être renommés quand tu as l’info.

## 5) Pipeline rédaction (GPT → Gemini)
Chaque page projet suit :
- Présentation courte
- Analyse visuelle factuelle
- ADN (5–7 mots)
- Galerie

## 6) Conseils
- 12 à 24 images max par projet (mieux vaut moins mais mieux)
- Compresse les images (1200–1800px largeur)
